# Docker Setup Guide for MacBook Pro

## Quick Start (3 Steps)

### Step 1: Get Your Discord Token

You need a Discord user token (not bot token) for the relay to work with Group DMs:

**Option A: From Browser (Easiest)**
1. Open Discord in browser (Chrome/Safari)
2. Open Developer Tools (Cmd+Option+I)
3. Go to Application → Local Storage → https://discord.com
4. Find the `token` key and copy the value (without quotes)

**Option B: From Desktop App**
1. Open Discord app
2. Press Cmd+Option+I to open DevTools
3. Go to Console tab
4. Type: `localStorage.getItem('token')`
5. Copy the token (without quotes)

⚠️ **IMPORTANT:** Never share this token! It gives full access to your account.

### Step 2: Create Environment File

Create a file named `.env` in the project root:

```bash
cd /Users/reidar/Documents/Lobster/Ine-bot
cp .env.docker.example .env
```

Edit `.env` and replace `your_user_token_here` with your actual Discord token:

```bash
DISCORD_USER_TOKEN=YOUR_TOKEN_HERE
```

### Step 3: Start Docker Services

**Option A: Start Everything (Ollama + ComfyUI + Bot)**
```bash
# Build and start all services
docker-compose -f .docker-compose.mac.yml up --build

# Or run in background (detached mode)
docker-compose -f .docker-compose.mac.yml up -d --build
```

**Option B: Start Services Separately (Recommended for Testing)**

First, start the AI services:
```bash
# Start just Ollama and ComfyUI
docker-compose -f .docker-compose.mac.yml up ollama comfyui

# Wait for them to be ready (see health checks below)
```

Then in another terminal, start the relay bot:
```bash
# Start the Discord relay bot
docker-compose -f .docker-compose.mac.yml up ine-relay
```

## Verification Steps

### Check Ollama is Running
```bash
curl http://localhost:11434/api/tags
```
You should see a list of available models.

### Check ComfyUI is Running
Open browser to: http://localhost:8188

You should see the ComfyUI interface.

### Check Bot Health
```bash
curl http://localhost:3001/health
```
You should see JSON with status "healthy".

3. Send: `@inebotten hei`

## Common Issues

### Issue: "Cannot connect to Docker"
**Fix:** Make sure Docker Desktop is running (check the whale icon in menu bar)

### Issue: "Invalid token"
**Fix:** 
- Make sure you copied the token correctly (no quotes)
- Token format should be like: `MTI5...xxxxxxxx...`
- Tokens expire - you may need to get a fresh one

### Issue: "Cannot build image"
**Fix:**
```bash
# Make sure you're in the right directory
cd /Users/reidar/Documents/Lobster/Ine-bot

# Clean and rebuild
docker-compose -f .docker-compose.mac.yml down
docker-compose -f .docker-compose.mac.yml up --build
```

### Issue: "Port already in use"
**Fix:** Something is using port 11434, 8188, or 3001
```bash
# Find what's using the port
lsof -i :11434

# Or use different ports by editing .docker-compose.mac.yml
```

### Issue: Apple Silicon (M1/M2/M3) Performance
Ollama will run slower on Apple Silicon Macs because:
- The docker image uses CPU mode (no GPU acceleration in Docker for Mac)
- For better performance, run Ollama natively instead of Docker

**Alternative: Run Ollama Natively**
```bash
# Install Ollama for Mac
brew install ollama

# Run Ollama natively (better performance on Apple Silicon)
ollama serve

# Then only run ComfyUI in Docker
docker-compose -f .docker-compose.mac.yml up comfyui
```

## Stopping the Services

```bash
# Stop all services
docker-compose -f .docker-compose.mac.yml down

# Or stop and remove volumes (deletes all data!)
docker-compose -f .docker-compose.mac.yml down -v
```

## Viewing Logs

```bash
# View all logs
docker-compose -f .docker-compose.mac.yml logs -f

# View specific service logs
docker-compose -f .docker-compose.mac.yml logs -f ine-relay
docker-compose -f .docker-compose.mac.yml logs -f ollama
```
